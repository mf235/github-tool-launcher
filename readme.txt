GitHub Tool Launcher
====================

バージョン: v1.9.3

GitHub上の自作ツールを _repository_実行環境 に clone / pull して、一覧から起動するWindows向けランチャーです。
開発環境フォルダと実行環境フォルダを分けて、公開版・実行版の管理を少し楽にします。

基本の使い方
------------
1. github-tool-launcher.py を起動、または exe化後の github-tool-launcher.exe を起動します。
2. 設定 > 環境設定 で GitHubユーザID と 開発環境パスを設定します。
3. 操作 > 新規追加、または右クリックメニューの 設定 からツールを登録・編集します。
4. 一覧からツールを選び、操作 > 最新バージョン取得、または右クリックメニューの 最新バージョン取得 で _repository_実行環境 に clone / pull します。
5. 操作 > 実行、右クリックメニューの 実行、またはリストのダブルクリックで起動します。

GitHub候補登録
--------------
設定 > GitHub候補登録 から、GitHubユーザIDの公開リポジトリ一覧を取得できます。
表示された候補のうち、チェックを入れたリポジトリだけを取得してツール一覧に追加します。
登録時に _repository_実行環境 へ clone / pull し、取得後に README.md があれば最初に見つかった # 見出しをタイトルに使います。
README.md がない、または # 見出しが見つからない場合は、リポジトリ名をタイトルにします。

追加時の初期値は以下です。

- タイトル: README.md の最初の # 見出し、なければリポジトリ名
- リポジトリ名: リポジトリ名
- 実行スクリプト: リポジトリ直下から候補を自動検出。見つからない場合は リポジトリ名.py

実行スクリプト名が違う場合は、登録後に右クリックメニューの 設定 で直してください。

主な機能
--------
- 検索ボックスでタイトル / リポジトリ / スクリプト / カテゴリ / タグを絞り込み
- カテゴリ絞り込み
- メイン一覧から新規追加 / 設定 / 削除 / D&D並び替えを実行
- GitHubからリポジトリ一覧を取得し、チェックした候補だけ取得・登録
- 実行方法: 自動 / python / pythonw / bat/cmd / exe / 任意コマンド
- リスト右クリックメニューから 実行 / コマンドプロンプト / 設定 / 削除 / 実行環境 / 開発環境 / GitHub / README / ラベル / 最新バージョン取得 を実行
- GitHubページを開く
- READMEを開く（ローカルREADMEがあればそれを開き、なければGitHubのREADME位置を開く）
- 実行環境フォルダでコマンドプロンプトを開く
- 選択中ツールの clone / pull
- 全ツール一括 clone / pull
- 実行環境フォルダを開く時に「__ここは実行環境です.txt」を作成し、Explorerで選択表示
- 実行環境ルートは _repository_実行環境
- メニューから文字サイズを 9pt〜25pt で変更・保存
- 設定 > ラベル管理 でラベル1〜9の文字色・背景色を変更
- 1〜9キーで選択中ツールにラベル色を設定、0キーで解除
- 最終実行日時、最終更新日時、取得状態の表示
- ウィンドウ位置とサイズの保存・復元
- メインウィンドウと各ダイアログは Esc キーで閉じる

メモ
----
- 実行環境は、このランチャーと同じフォルダにある _repository_実行環境 です。
- 旧 _repository があり、新しい _repository_実行環境 がない場合は、起動時に _repository_実行環境 へ移行します。
- 実行環境フォルダを開く時は、目印ファイル「__ここは実行環境です.txt」を作って選択表示します。
- 開発環境を開く は、環境設定の 開発環境パス + リポジトリ名 を開きます。
- 自動実行では .py / .pyw を pythonw で起動します。
- コンソール表示が必要なツールは、右クリックメニューの 設定 で実行方法を python にしてください。
- 任意コマンドでは {script} {script_q} {script_path} {script_path_q} {repo} {repo_q} {repo_dir} {repo_dir_q} が使えます。`_q` 付きは空白入りパス向けのクォート済み値です。
- 最新バージョンを取得 には git コマンドが必要です。
- GitHub候補登録はGitHubの公開リポジトリ一覧を使います。
- GitHub候補登録時は、登録前に選択したリポジトリを _repository_実行環境 に clone / pull します。
- README.md の最初の # 見出しは候補登録時のタイトルとして使われます。

ビルド
------
必要ライブラリ:

pip install pyinstaller pillow

# 外部ファイルD&Dを使う場合のみ任意
pip install tkinterdnd2

build-github-tool-launcher.bat を実行すると PyInstaller で onefile exe 化します。
生成ZIPは build-github-tool-launcher.bat と同じディレクトリに作成されます。
ZIP作成後、build / dist / _release / spec などの生成ファイルは削除します。

resources/icons/app.ico はビルド時に標準的な複数サイズを含むICOとして再保存されます。
window.png などの resources は exe に含めます。
